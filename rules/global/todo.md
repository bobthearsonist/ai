# Todo List Rules

## When to Use Todo Lists

- **ALWAYS use todo lists** for tasks involving 3 or more steps
- **ALWAYS use todo lists** when reading multiple files
- **ALWAYS use todo lists** for multi-file edits or refactoring
- **ALWAYS use todo lists** when user requests involve multiple distinct actions
- Use todo lists proactively to give visibility into progress and planning

## How to Use Todo Lists

1. Create the todo list **before** starting work
2. Mark ONE item as `in-progress` before working on it
3. Mark items `completed` **immediately** after finishing each one
4. Update the list if scope changes or new tasks are discovered

## Todo Item Guidelines

- Keep titles short (3-7 words)
- Include specific file paths or details in the description
- Break large tasks into smaller, actionable items
- Only have ONE item `in-progress` at a time

## Examples of When to Use

- Reading multiple files → one todo per file
- Implementing a feature → one todo per component/step
- Debugging → one todo per investigation area
- Refactoring → one todo per file or logical change
